# Règles Communes

## Gestion de projet

- Daily tous les matins vers 9h - 2 minutes/dev :
  - Point d'avancement
  - Difficultés rencontrées

## Standards de qualité

- Pas de commit sur la branche Main
- Commits réguliers (- respectant la nomenclature contenant l'id de l'issue) avec la nomenclature suivante :
  `Numéro_de_l'us-fonctionalité_développée`

- Chaque fonctionnalité est développpée sur une branche unique à partir de la branche dev avec la nomenclature suivante :
  `Nom_du_dev-fonctionnalité`

- Les merges sur dev sont réalisés une fois la fonctionnalité terminée depuis les branches fonctionnalités.
